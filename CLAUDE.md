# Project Constitution

## リポジトリ
https://github.com/kumagaicorp/NOREN

## 目的
このリポジトリは、AI開発チーム運用アーキテクチャ仕様に準拠して運用する。

## 原則
- 仕様の規範（`docs/spec.md`）を最優先で遵守する。
- 変更は小さく、差分はレビュー可能な単位に分割する。
- 重要な判断は必ず `docs/ADR/` に記録する。
- テスト可能な変更はテストを追加し、実行結果を残す。

## 禁止事項
- 仕様に反する構造変更やルールの破棄
- 目的不明な依存追加
- 機密情報のコミット

## Git ワークフロー（AI必須）

### 作業開始時
```bash
git pull origin main
```

### 作業終了時
```bash
git add .
git commit -m "変更内容の要約"
git push origin main
```

### コミットルール
- 変更は小さく、レビュー可能な単位でコミットする
- コミットメッセージは変更内容を簡潔に記述する
- 問題発生時にロールバックできるよう、こまめにコミットする

### ロールバック（問題発生時）
```bash
# 直前のコミットに戻す
git revert HEAD

# 特定のコミットに戻す
git revert <commit-hash>
```

## 作業フロー
1. **git pull** で最新を取得する
2. `docs/TASKS.md` の順序に従って作業する
3. タスク完了ごとに変更ファイル/コマンド/テスト結果/次タスクを記録する
4. 影響範囲が大きい場合は事前にADR化する
5. **git commit & push** で変更を保存する
