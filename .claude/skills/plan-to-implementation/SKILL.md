# Plan to Implementation

Cursorの計画書をClaude Codeが実装可能な手順に変換する。

## 入力

- 実装計画（docs/TASKS.md）
- WBS（マイルストーン/タスク）
- 受入基準
- 影響範囲
- ファイル候補

## 出力

- タスク順序
- 変更ファイル一覧
- 実装チェックリスト
- 実行コマンド
- 完了条件

## ワークフロー

1. docs/TASKS.md を読み込み、対象タスクを特定する
2. タスクの Goal / Files to touch / Implementation steps を抽出する
3. 変更ファイル一覧を作成する
4. 各ファイルの変更内容を明確化する
5. 実行コマンドを洗い出す
6. テスト方法と完了条件を整理する
7. チェックリストを作成する

## 出力フォーマット

```markdown
## タスク: [タスク名]

### 変更ファイル
- path/to/file1
- path/to/file2

### 実装手順
1. [手順1]
2. [手順2]

### 実行コマンド
[コマンド]

### テスト
- [テスト内容]

### 完了条件
- [条件1]
- [条件2]
```

## 受入条件

- 各タスクに「変更箇所」「完了定義」「テスト」が含まれている
- 実行コマンドが具体化されている
- 依存順序が考慮されている
