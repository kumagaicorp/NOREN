# UI/UX Specification Translator

ラフ・スクショ・メモをUI/UX仕様に変換する。

## 入力

- 紙ラフ／文章
- 既存UIのスクリーンショット（参考用）
- MVP条件

## 出力

- 画面概要
- 情報設計
- レイアウト構造（文章）
- 操作フロー
- 状態一覧
- UX注意点

## 禁止事項

- コード生成
- デザインの無断コピー（表層模倣）
- スタイルの直接コピー
- 実装前提の技術決定

## スクリーンショット解析方針

「構造・情報設計・操作意図」のみ抽出:

1. **レイアウト構造** - 要素の配置、グループ化
2. **情報の優先順位** - 視覚的ヒエラルキー
3. **操作フロー** - ユーザーの行動経路
4. **状態変化** - 初期/ローディング/空/正常/エラー

色・余白・フォントは tokens / shadcn/ui 前提で再解釈する。

## UX仕様書 標準フォーマット

```markdown
# 画面名

## 目的
[この画面が解決する課題]

## 参考にした入力
- 紙ラフ: 有/無
- スクリーンショット: 有/無（説明）

## 抽出した意図
[参考UIから何を学んだか]

## 採用要素 / 不採用要素
- 採用: [要素と理由]
- 不採用: [要素と理由]
- 再解釈: [要素と理由]

## 情報設計（優先度）
1. [最重要情報]
2. [重要情報]
3. [補足情報]

## レイアウト構造
[文章でレイアウトを説明]

## コンポーネント（抽象名）
- [コンポーネント1]
- [コンポーネント2]

## 操作フロー
1. [ステップ1]
2. [ステップ2]

## 状態一覧
- 初期: [説明]
- ローディング: [説明]
- 空: [説明]
- 正常: [説明]
- エラー: [説明]

## a11y配慮
- [配慮点1]
- [配慮点2]

## 受入基準（Given/When/Then）
- Given [前提条件], When [操作], Then [期待結果]
```

## ワークフロー

1. 入力を確認する（テキスト、ラフ、スクショ）
2. ユーザーの意図を整理する
3. スクリーンショットから構造・情報設計・操作意図を抽出する
4. 採用要素 / 不採用要素 / 再解釈要素を明確にする
5. UX仕様書を標準フォーマットで作成する
6. docs/UX/ に配置する

## 受入条件

- コード生成をしていない
- スタイルを直接コピーしていない
- 採用/不採用/再解釈が明示されている
- tokens / shadcn/ui 前提で再解釈している
- 標準フォーマットに従っている
